# BirdCLEF Autonomous Meta-Agent

Autonomous research agent for the **BirdCLEF 2026** competition (APA course project). An LLM **researcher** proposes experiments; a separate **coder** implements them; subprocesses train and evaluate models; results are stored in **experiment memory** and drive the next iteration. The pipeline searches cheaply on a data subset, picks a **CNN vs Perch** tournament winner, then runs full-data training and hardcoded pseudo-labelling for Kaggle submission.

This repository is structured for **D1 (GitHub repository)** deliverables: source code, configuration, dependency file, experiment logs, and instructions so a reviewer can clone, install, start a local LLM, and run the agent with a short documented sequence.
See our agent demonstration video here: https://www.youtube.com/watch?v=ZbqlASTRl54

In order for our grader to be able to see that the agent improves overtime, we turned down the CNN track iterations of the model to a minumum and also included an older EDA run of the agent, so the different phases are still visible but CNN wthout a lot of improvement due to the limited number of experiments. The Perch agent track runs on a normal number of iterations that likely will finish with more powerful resources (at least the architecture decision phases, maybe not the last final training and pseudo-labelling step). Please note that while producing our own submissions we often tweaked around these iterations, to see how much the agent improves with different iterations/experiment numbers.
Short notice: A macro averge precsision of around 0.290 resulted in a score of 0.869 (MACRO AUC leaerboard score) and macro average precision of around 0.30 produced our best result of 0.88.

## Requirements

- **Python 3.11.x** (tested with **3.11.15**). TensorFlow and several audio/ML wheels expect 3.11; other versions often break venv creation or `pip install`.
- **Ollama** (or another provider supported by `src/llm_client.py`) for local LLM inference  
- **Disk**: BirdCLEF data (~tens of GB) + caches/embeddings + logs  
- **GPU**: recommended for CNN training and Perch head training (CPU-only is possible but very slow)

## Quick start (reviewer checklist)

From the repository root:

### 1. Dataset

Download the [BirdCLEF 2026](https://www.kaggle.com/competitions/birdclef-2026) competition data and place it under `data/` (standard Kaggle layout), keeping this kind of structure perserved after the kaggle download:

- `data/train.csv`, `data/taxonomy.csv`  
- `data/train_audio/` (focal training clips)  
- `data/train_soundscapes/`, `data/train_soundscapes_labels.csv` 
etc. 



### 2. Create virtual environment (setup script)

Use **Python 3.11** explicitly when creating the venv:

```bash
python scripts/setup_project.py
```
The script creates `.venv` at the repo root and installs `requirements.txt`.

**If setup fails** (venv not created, TensorFlow/pip errors): you are likely on the wrong Python version. Install Python 3.11 (or the indiacted Python version from the error code), then run the setup script again with that interpreter (e.g. `python3.11 scripts/setup_project.py`).

### 3. Activate the virtual environment

Paths depend on your **current working directory**:

```bash
# From repo root (example, adapt path and way depending on your local setup)
source .venv/bin/activate       
```


### 4. Install and verify the local LLM

The default configuration uses **Ollama** with **`qwen2.5-coder:7b`** (fast coder model used for both researcher and coder in our runs). **You must pull this model before running the agent:**

```bash
ollama pull qwen2.5-coder:7b
```

To use a different model, edit `configs/agent_config.json` (`llm.model`, `researcher.model`, `llm_researcher.model`) and pull that tag in Ollama (some models may not work as good as others).

### 5. Run the agent (single command)

With the venv activated and Ollama running (adapt it to produce a log file if you do not want a truncated terminal log as it becomes really large):

```bash
python src/meta_agent.py --config configs/agent_config.json
```

log command:
```bash
python src/meta_agent.py --config configs/agent_config.json 2>&1 | tee -a logs/meta_agent_terminal.log
```

Equivalent if `python` in the venv is 3.11:

```bash
python3 src/meta_agent.py --config configs/agent_config.json
```



**Shorter documented sequence:**

```bash
python3.11 scripts/setup_project.py
source .venv/bin/activate
ollama pull qwen2.5-coder:7b
python src/meta_agent.py --config configs/agent_config.json
```

(Place competition files in `data/` before step 5.)

## Architecture overview (CNN + Perch)

The meta-agent implements a **staged tournament** designed for cheap search, reliable local ranking, and one expensive full-data path.

```
Phase 0 â€” EDA (eda_agent.py)
    â””â”€â”€ eda_brief.txt fed to researchers

Phase A â€” Tournament (sample / cheap data, ~1500 clips)
    â”śâ”€â”€ Track: CNN (mel-spectrogram, model from scratch)
    â”‚     1a Explore architectures
    â”‚     1b Refine best architecture
    â”‚     1c Augmentation search
    â””â”€â”€ Track: Perch (frozen Google Perch embeddings, agent searches heads only)
          1a / 1b / 1c (same structure)

    â†’ Compare tracks on macro Average Precision (labelled train_soundscapes)
    â†’ tournament_results.json â†’ global winner

Phase B â€” Finalize winner only
    1d Full-data training (all focal samples, winning aug + arch)
    1e Pseudo-label unlabelled soundscapes (fixed thresholds, no LLM)
        â†’ final weights + submission/ bundle
```

**Why two tracks?** CNN satisfies from-scratch exploration; Perch uses a strong pretrained bird vocalisation encoder. They compete under the same local metric; only the winner gets full train + pseudo.

**Why 1500 samples?** Balance between stable architecture ranking and fast iteration. Full data is used only after the tournament.

**Why macro AP?** Local macro AUC often mis-ranked Perch vs CNN relative to Kaggle; macro AP on labelled soundscapes gave better **ordering** for the researcher loop (see project report).

**Roles**

- **Researcher LLM** â€” reads `experiment_memory.jsonl`, proposes next specs  
- **Coder LLM** â€” writes training code from spec only (no memory)  
- **Executor** â€” `code_executor.py` runs scripts in subprocesses  
- **Evaluator** â€” validation metrics + `soundscape_evaluator.py` for benchmark scores

## `meta_agent.py` â€” central hub

`src/meta_agent.py` is the **only entry point** you need for the full course pipeline. It:

1. Loads `configs/agent_config.json`  
2. Prints a run roadmap (`run_log.py`)  
3. Runs **Phase 0 EDA** (`ensure_meta_eda_before_tracks` â†’ `eda_agent.py`)  
4. Dispatches **staged tournament** (`run_staged_tournament_phase`): for each track in `track_order`, launches stage 1a/1b/1c via subprocesses or imports (`cnn_staged.py`, `perch_agent.py`)  
5. Picks **global winner** and saves `logs/meta_agent/tournament_results.json`  
6. If `tournament.auto_finalize`, runs **1d + 1e** for the winner only (`run_staged_finalize_winner`)  
7. Bundles Kaggle artifacts under `submission/` when configured  

Supporting modules (called by meta-agent or track subprocesses, not run directly for the main demo):

| Module | Role |
|--------|------|
| `eda_agent.py` | Dataset exploration and brief |
| `cnn_staged.py` | CNN 1a / 1b / 1c / 1d / 1e |
| `perch_agent.py` | Perch explore / refine / final retrain |
| `memory.py` / `perch_memory.py` | Append-only experiment memory for the researcher |
| `experiment_tracker.py` | Timeline JSON + progress plots |
| `llm_client.py` | Ollama / OpenAI-compatible API |
| `augmentation.py`, `aug_researcher.py`, `cnn_aug_researcher.py` | Aug search |
| `cnn_pseudo.py`, `perch_pseudo.py` | Stage 1e pseudo-labelling |
| `soundscape_evaluator.py` | Macro AP / AUC on labelled soundscapes |

## Troubleshooting

| Issue | What to do |
|-------|------------|
| `setup_project.py` / venv fails | Use **Python 3.11** (`python3.11 scripts/setup_project.py`), then re-activate `.venv` |
| Wrong `python` after activate | Use `which python` â€” should point inside `.venv` |
| Ollama connection errors | Run `ollama serve`; confirm `ollama list` shows `qwen2.5-coder:7b` |
| Missing data errors | Ensure `data/train.csv`, `data/train_audio/`, soundscape labels/audio exist |
| TensorFlow errors on import | Confirm venv was created with Python 3.11, not 3.12+ |
| Run takes many hours - Longest running times expected in full training in the end and pseudo labelling (10599 files) |
